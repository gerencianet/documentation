Interpretando Erros na API

Esta página tem como objetivo apresentar o "Histórico de Requisições", que permite ao integrador visualizar todas as requisições realizadas por sua aplicação à API Gerencianet. Assim, você vai aprender a interpretar os retornos da API e, claro, corrigir possíveis erros de validação de dados ou outros similares e identificar consumos/requisições realizadas por sua aplicação aos nossos webservices.


Aprendendo a utilizar o "Histórico de Requisições" da API Gerencianet#

A API efetua validações dos dados contidos na requisição que o sistema integrado envia a nossos webservices. Dessa forma, conforme o endpoint que seu sistema está tentando consumir, podem ser retornados erros em determinadas situações. Isso também se aplica se você utiliza um sistema ou módulo que tenha integração com a Gerencianet.

Para acessar o "Histórico de Requisições", siga os passos:

  • Efetue login em sua conta Gerencianet;
  • Acesse API > Minhas Aplicações > Sua Aplicação;
  • Escolha o ambiente utilizado (produção ou desenvolvimento) e clique em Histórico de Requisições (veja onde).

Exemplo 1 - Criar transação/cobrança#

Ao enviar um POST para a rota /v1/charge, o integrador estará criando uma cobrança (ou "transação") com o status new, ou seja, a cobrança foi gerada e está aguardando a definição da forma de pagamento.

Ao criar uma transação, podem ser definidos, por exemplo: nome do item, valor, se é item de marketplace, se tem frete, informar uma URL de notificação, etc.

Exemplo 2 - Associar à forma de pagamento#

Ao enviar um POST para a rota /v1/charge/:id/pay, onde :id é o charge_id da transação desejada, o integrador estará associando a cobrança à uma forma de pagamento.

(+) Códigos de status HTTP (2xx, 3xx, 4xx e 5xx)#

A Gerencianet utiliza respostas HTTP para indicar sucesso ou falha nas requisições. Comumente, quando retornamos respostas com status 2xx significa que houve sucesso na requisição; status 3xx indicam redirecionamento; status 4xx indicam falhas no envio de dados por parte do cliente; status 5xx indicam erros internos de servidor.

Caso queira, confira neste link uma tabela apresentando os códigos HTTP das respostas mais comuns, bem como suas explicações e soluções.


(+) Aba "Histórico de Notificações"#

Este recurso está disponível na API de sua conta Gerencianet e permite visualizar os POSTs que a Gerencianet dispara para a URL de notificação definida pelo integrador. Saiba mais.
